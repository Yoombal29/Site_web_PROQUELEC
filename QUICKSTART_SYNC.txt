# ⚡ QUICK START - Synchronisation Bidirectionnelle

## 🎯 Objectif (1 minute)
Remplacer AdminPageEditor par AdvancedPageEditorSync pour synchroniser 4 modes d'édition.

---

## 📋 4 étapes (15 minutes)

### **1️⃣ Valider le système**
```bash
node validate-sync.cjs
```
Expected: ✓ Tous les fichiers existent et syntaxe OK

### **2️⃣ Lancer les tests**
```bash
npm test -- AdvancedPageEditorSync.test.tsx
```
Expected: ✓ 30+ tests passent

### **3️⃣ Intégrer dans AdminPagesPanel.tsx**

```typescript
// AVANT
import AdminPageEditor from './AdminPageEditor';

// APRÈS
import { AdvancedPageEditorSync } from './AdvancedPageEditorSync';
```

Puis remplacer le composant:
```typescript
// AVANT
<AdminPageEditor pageId={editingPage.id} />

// APRÈS
<AdvancedPageEditorSync 
  pageId={editingPage.id}
  initialPage={editingPage}
  onSave={() => {
    // Optional: close dialog, refresh, etc
    setEditingDialogOpen(false);
  }}
/>
```

### **4️⃣ Tester dans le navigateur**

```bash
npm run dev
```

- Aller à `/admin/pages`
- Cliquer "Edit" sur une page
- Voir les 5 onglets: HTML, JSON, Form, Visual, Preview
- Modifier du texte dans HTML → Voir JSON se mettre à jour
- Modifier le JSON → Voir HTML se mettre à jour  
- Vérifier auto-save après 1s
- Vérifier site public se met à jour <500ms

## ❌ Problèmes courants

### Erreur: "AdvancedPageEditorSync not found"
```
→ Vérifier import correct
→ Vérifier fichier existe: src/components/admin/AdvancedPageEditorSync.tsx
```

### Onglets ne se synchronisent pas
```
→ Vérifier console pentru erreurs
→ Lancer: npm test -- AdvancedPageEditorSync.test.tsx
```

### Auto-save ne sauvegarde pas
```
→ Vérifier API endpoint exists: PUT /api/pages/:id
→ Vérifier Network tab pour les requêtes
```
